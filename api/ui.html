This project predicts greenhouse climate and evaluates control strategy.

The model takes a recent time window of greenhouse and outdoor conditions and predicts the next greenhouse state. Around that core forecast, the system compares alternative model architectures, shows feature importance and timestep influence, and runs model predictive control simulations to test how different control objectives affect performance.

What the model predicts

The main prediction targets are greenhouse air temperature, CO2 concentration, and humidity deficit. These are the core outputs used for monitoring and control decisions.

What the rest of the system adds

Model comparison shows which architecture performs best, Explainability shows which inputs drove a forecast, and MPC simulates how control settings influence future greenhouse behavior.

What makes this useful

This system helps answer practical greenhouse questions: what conditions are likely next, which trained model is most reliable, what signals drove the forecast, and which control policy looks most effective before deployment.

  • Forecast target greenhouse conditions from recent sensor and weather history.
  • Compare trained architectures using ranked error metrics.
  • Simulate control actions before applying them to a real greenhouse setting.
  • Expose explainability views for trust, validation, and debugging.
